02 / CONCEPTS

01OPENCLAW 是什么？

一个运行在本地 Mac 上的 多 Agents 协作平台，通过飞书与用户交互。

关键词：7×24小时运行 · 定时任务 · 长期记忆 · 多Agents协作

🎯 解决什么问题？

  • ChatGPT 用完就关机 → OpenClaw 7×24 待命
  • 每次都要重新解释背景 → 记住你的习惯和偏好
  • 只能手动触发任务 → 定时自动执行
  • 一个 Agent 什么都会一点 → 专业 Agent 团队协作

🔄 传统 AI 助手 vs OpenClaw

DIMENSIONCHATGPT/CLAUDEOPENCLAW
运行方式用时打开7×24 持续运行
定时任务不支持心跳机制原生支持
记忆会话级别持久化存储
多Agents单 Agent多 Agents 协作
交互方式网页/APP飞书/微信/命令行

03 / CORE FEATURES

01心跳机制 HEARTBEAT

💓 什么是心跳？

OpenClaw 的定时任务触发器，让它可以主动执行任务。

⏰ 工作流程

  1. 配置心跳：设定执行间隔（如每天9点）
  2. 触发执行：到达时间自动执行
  3. 执行动作：获取新闻、监控股票等
  4. 汇报结果：通过飞书推送

📁 记忆文件结构

openclaw/
├── identity/           # 你的身份定义
│   └── default.md     # 主身份文件
├── soul/              # 对话风格
│   └── default.md     # 说话方式
├── memory/            # 长期记忆
│   ├── 私聊.md        # 私聊记忆
│   ├── 主群.md        # 主群记忆
│   └── 量化群.md      # 量化群记忆
└── openclaw.json      # 核心配置

⚠️ 关键原则

  • 记忆要"写"出来：重要内容存到文件，不要只靠对话
  • 群聊单独记忆：默认只记私聊，需为每个群建独立文件
  • 定期整理：memory 文件会越来越大，需要定期精简

06 / PITFALLS

01方法论：如何培养 OpenClaw？

培养 OpenClaw 的本质：不是下命令，而是建立机制。

🛡️ 原则一：不要让模型改核心配置

  • 危险操作：通过对话让模型修改 openclaw.json
  • 原则：核心配置要手动修改

💾 原则二：定时备份核心文件

  • 需要备份：openclaw.json、memory/、identity、soul
  • 建议：配置心跳任务，让它自己定期备份

📝 原则三：写到配置里，不要靠记忆

  • 问题：让它"记住"记账位置，下次又忘了
  • 解决：写到 tools 配置里

08 / PRACTICE

02日常记账

💰 完整流程

  1. 用户说："今天吃饭花了128元"
  2. OpenClaw 解析：金额=128，类别=餐饮
  3. 调用 Tool：写入飞书多维表格
  4. 确认回复："已记录：餐饮 128元"
  5. 月底汇总：生成消费报表

📋 飞书表格结构

| 日期 | 金额 | 分类 | 备注 |
|------|------|------|------|
| 3/12 | 128  | 餐饮 | 午餐  |
| 3/12 | 45   | 交通 | 地铁  |
