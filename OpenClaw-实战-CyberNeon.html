02 / CONCEPTS

01OPENCLAW 是什么？

一款开源、自托管 AI Agent 框架，像拥有"主动性"的数字员工，能操作电脑、浏览网页、处理文件，通过飞书/Telegram/微信执行指令。（可运行在 Mac/Linux/云服务器，我选择 Mac Mini 是因为 7×24 运行、静音、省电）

关键词：7×24小时运行 · 定时任务 · 长期记忆 · 多Agents协作

🎯 解决什么问题？

  • ChatGPT 用完就关机 → OpenClaw 7×24 待命
  • 每次都要重新解释背景 → 记住你的习惯和偏好
  • 只能手动触发任务 → 定时自动执行
  • 一个 Agent 什么都会一点 → 专业 Agent 团队协作

🔄 传统 AI 助手 vs OpenClaw

03 / CORE FEATURES

01定时任务 & 心跳机制

两者都像"闹钟"，让 AI 助手自动干活，但逻辑完全不同：

⏱️ 定时任务 (Cron)

精准的"执行机器"

  • 触发：精准时间点/间隔
  • 特点：独立运行，无记忆
  • 打扰：到点就执行
  • 用途：定点发报、定时备份

💓 心跳机制 (Heartbeat)

会"思考"的"巡视员"

  • 触发：定期巡查，判断时机
  • 特点：带记忆，感知上下文
  • 打扰：有事才通知
  • 用途：持续监控、批量提醒

🤔 如何选择？

需要准点执行？→ Cron  |  需要批量检查？→ Heartbeat  |  追求极致效率？→ 两者结合

06 / PITFALLS

01方法论：如何培养 OpenClaw？

培养 OpenClaw 的本质：不是下命令，而是建立机制。

🌟 原则零：别把他当工具，把他当人（们）

  • 说人话：让他读、让他思考、让他努力、让他写、让他说他的理解并跟你对齐
  • 说到他心里去：了解他的底层机制
    ❌ "你记住了吧"
    ✅ "你写到自己的记忆里了吗，给我展示一下现在的结构和内容"

🛡️ 原则一：不要让模型改核心配置

  • 危险操作：通过对话让模型修改 openclaw.json
  • 原则：核心配置通过 openclaw 命令 修改，进阶可手动修改配置文件

💾 原则二：定时备份核心文件

  • 需要备份：openclaw.json、memory、identity、soul
  • 建议：配置心跳任务，让它自己定期备份

📝 原则三：不要相信口头，要写入配置并定期检查

  • 问题：让它口头"记住"记账位置，下次又忘了
  • 解决：写到 tools 配置里，并定期检查配置是否生效

06 / PITFALLS

02常见坑与解决方案

🚫 飞书相关

  • 无法拉群 → 用企业版飞书
  • 机器人无响应 → 检查权限：im:message, im:resource, im:chat
  • 消息发不出去 → 确认应用已开通对应群聊权限

⏰ 定时任务

  • 任务不执行 → 简化任务，一次只做一件事
  • 执行结果不对 → 检查 HEARTBEAT.md 格式是否正确
  • 间隔太短 → 建议 cron 间隔不少于 5 分钟

🧠 记忆问题

  • 群聊没记忆 → 在 memory/ 下建群聊专属文件
  • 教过又忘了 → 写入配置文件，不要只靠对话
  • 记忆文件太大 → 定期整理，精简内容

🔧 能力缺失

  • 无法理解图片 → 安装 minimax-understand-image skill
  • 无法搜索 → 安装 last30days-lite skill
  • 能力不够用 → 去 ClawHub 安装更多 skills

⚙️ 配置方式（CLI）

# 添加定时任务
openclaw cron add --name "每日汇总" --cron "0 9 * * *" --session isolated --message "请汇总今日要点"

# 每天早上9点
openclaw cron add --name "新闻播报" --cron "0 9 * * *" --session default --message "请播报今日新闻"

# 工作日9点（A股报告）
openclaw cron add --name "A股报告" --cron "0 9 * * 1-5" --session default --message "请生成今日A股报告"

# 每30分钟（盘中监控）
openclaw cron add --name "盘中监控" --cron "*/30 * * * *" --session isolated --message "请监控持仓股票"
